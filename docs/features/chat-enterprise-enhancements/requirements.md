# 聊天增强（企业向）需求说明

## 1. 背景与目标

在现有 Next.js 流式聊天（SSE、断点续传、NextAuth）基础上，按**生产可用、可审计、可水平扩展**的标准补齐：上下文与单条配额、草稿分级持久化、服务端校验与纵深限流、续传会话外置存储与缓冲上限。

**决策选项串（已确认）**：`1C, 2C, 3C, 4B, 5C, 6C, 7B, 8C`

---

## 2. 范围

| 包含 | 不包含（除非另行立项） |
|------|------------------------|
| Token/上下文预算策略（总上下文 + 单条） | 多租户计费与按套餐动态配额 |
| 输入草稿：默认短生命周期 + 可选长期记住 | 跨设备端到端加密同步 |
| 服务端消息校验与共享常量 | 网关层 WAF 规则定制 |
| 用户 + IP 双层限流 | 全球分布式限流（多区域统一计数） |
| 续传会话：Redis（或等价）+ TTL | 自研分布式会话存储 |

---

## 3. 功能需求

### 3.1 上下文与单条 Token/字符预算（1C）

- **总上下文**：对「发给模型」的合并上下文设定上限（可基于近似 token 或字符；实现需与现有模型调用链对齐）。
- **单条**：对用户单条输入与助手单条输出分别设定上限（字符或 token 策略与总上下文一致并文档化）。
- **目的**：同时覆盖「历史堆满」与「单条炸弹」类攻击与故障面。

### 3.2 超出预算时的行为（2C）

- **先裁剪**：在服务端按约定顺序尝试裁剪（例如优先裁剪较早历史、或摘要占位；具体顺序在实现阶段在架构文档中固定）。
- **仍超则拒绝**：裁剪后仍超过预算的，返回明确错误码与可读提示，不静默失败。
- **可审计**：裁剪或拒绝的原因应可被记录（日志或指标，至少一种）。

### 3.3 输入草稿持久化（3C）

- **默认**：草稿使用短生命周期存储（与「同浏览器会话」语义一致，如 `sessionStorage` 或等价）。
- **可选「记住草稿」**：仅当用户显式开启时，将草稿同步到长期存储（如 `localStorage` 或带前缀 key），并说明数据保留与清理策略（如登出清空、可选 TTL）。

### 3.4 草稿隔离（4B）

- 草稿按 **conversationId / threadId**（或项目中等价的会话标识）分 key 存储，避免多会话串稿。
- 若产品仅单会话入口，实现可退化为单 key，但数据结构应预留按线程扩展。

### 3.5 服务端消息校验（5C）

- **与前端一致的上限**：条数、单条长度等通过**共享常量或共享校验模块**维护，服务端为最终裁决。
- **语义校验**：禁止未知 `role`、禁止空消息等非法组合；超限返回 **400** 及一致的错误结构。
- **原则**：任何校验逻辑以服务端为准，前端仅作体验优化。

### 3.6 限流（6C）

- **用户维度**：对已登录用户（如 `guestId` / `userId`）进行请求或令牌桶类限流，防止账号级滥用。
- **IP 维度**：并行或叠加 IP 限流，缓解共享出口、未鉴权旁路（若未来存在）等场景。
- **响应**：限流时返回 **429**（或项目约定状态码）及可区分「用户限流」与「IP 限流」的提示或子码（实现时二选一，需一致）。

### 3.7 流会话与续传存储（7B）

- **部署假设**：支持多实例 / Serverless 水平扩展；续传状态**不得**依赖单机内存为唯一真相。
- **实现**：使用 **Redis（或等价外置存储）** 保存 `streamId` 关联的会话元数据、游标与可续传缓冲；设置 **TTL**，与业务上「流结束后的最长保留时间」一致。
- **身份**：续传请求须校验与会话绑定的用户标识（与现有 `guestId` 校验对齐），防止越权。

### 3.8 续传缓冲上限（8C）

- **条数上限**：单会话缓冲的最大事件条数（超过则丢弃最旧或拒绝新事件，策略需固定并文档化）。
- **字节上限**：单会话缓冲的总字节上限（防止单条极大 SSE 撑爆内存或 Redis）。
- **双上限同时生效**：任一触发均需有明确行为（拒绝写入、或裁剪、或结束流并记录）。

---

## 4. 非功能需求

- **可用性**：Redis 短时不可用时，续传路径应有降级（例如返回明确错误，引导用户重新发起对话），避免无限重试。
- **可观测性**：建议对裁剪次数、拒绝次数、限流命中、续传失败率打点或打日志（最小实现至少一种）。
- **安全**：敏感信息不全量写入客户端长期存储；登出时清理与用户相关的草稿 key（若适用）。

---

## 5. 验收要点（建议）

- [ ] 总上下文与单条超限路径均有测试用例或手动清单。
- [ ] 服务端校验无法被「仅改前端」绕过。
- [ ] 双实例模拟下续传仍可从 Redis 恢复（或文档说明本地仅单实例）。
- [ ] 缓冲触达条数/字节上限时行为符合 3.8 文档。
- [ ] 用户 + IP 限流在压测或脚本下可观测。

---

## 6. 依赖与环境

- **Redis（或托管 Redis）**：生产级续传必备；本地开发可用 Docker Compose 或云托管实例。
- **环境变量**：连接串、TLS、DB 索引等由部署文档列出（实现阶段补充）。
